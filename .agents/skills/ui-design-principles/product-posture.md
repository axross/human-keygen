# Product Posture

Human Keygen is a practical generator, not a marketing landing page. The first screen should help the user create a layout-aware password immediately.

## First-Viewport Hierarchy

The generator is the product. Brand and explanation support the controls instead of replacing them.

**Guidelines:**

- MUST make the generator the primary first-viewport experience.
- MUST NOT replace the app with a hero-only marketing page.
- MUST present layout selection, generation controls, generated password, and copy action as the main hierarchy.
- MUST avoid navigation structures that hide the generator behind another page.
- SHOULD keep explanatory copy short and attached to the controls it clarifies.
- SHOULD make the character-pool explanation scannable without crowding the generated password.

## User Mental Model

The UI should make the same-position-key rule understandable without turning the app into a technical whitepaper.

**Guidelines:**

- MUST explain that characters are included only when QWERTY and the selected layout share the same physical key and layer.
- MUST avoid implying that the app stores passwords, manages accounts, or acts as a vault.
- SHOULD show enough layout and pool information for trust-building inspection.
- SHOULD keep advanced details secondary to generation and copy tasks.
