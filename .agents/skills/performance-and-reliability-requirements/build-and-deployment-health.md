# Build and Deployment Health

The app is deployed as Cloudflare Workers Static Assets through a static/prerendered TanStack Start build. Build changes should preserve that simple deployment path.

## Build Health

Build failures often catch TypeScript, route, and bundling regressions that tests do not.

**Guidelines:**

- MUST run `npm run build` after route, config, dependency, TypeScript signature, Vite, Tailwind, or Wrangler changes.
- MUST keep `src/routeTree.gen.ts` generated by TanStack tooling rather than hand-authored.
- MUST keep build output free of generated passwords or user-specific state.
- SHOULD verify that prerendered output still serves the root generator route after routing changes.

## Cloudflare Workers Static Assets

Wrangler config owns asset deployment behavior and should remain boring.

**Guidelines:**

- MUST preserve `assets.directory` alignment with the TanStack Start build output unless the build output intentionally changes.
- MUST preserve `send_metrics: false` unless telemetry has explicit product approval.
- MUST update `docs/cloudflare-deployment.md` when deployment commands or Workers Builds settings change.
- SHOULD run the documented Wrangler dry-run path when deployment config changes and the environment allows it.
